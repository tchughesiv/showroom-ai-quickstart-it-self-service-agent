= IT Self-Service Agent Quickstart Showroom

Lab content for the *Automate IT Processes with IT Self-Service Agents* quickstart — AI-driven IT process automation using Red Hat OpenShift AI, Llama Stack, LangGraph, and MCP servers.

== Content Repo

This Antora-based showroom provides hands-on lab instructions for:

* Understanding the IT self-service agent architecture (assumes a provisioned Demo Mode environment)
* Completing the laptop refresh flow via the test email UI (Demo Mode uses Greenmail for in-cluster test mail)
* Going further with extensibility, evaluations, and tracing (see lab modules)

The install and quickstart code lives in https://github.com/rh-ai-quickstart/it-self-service-agent[rh-ai-quickstart/it-self-service-agent].

== Pinning quickstart GitHub links

Lab pages resolve quickstart URLs from AsciiDoc attributes in `content/antora.yml`:

* `quickstart_repo_url` — repository base (typically unchanged)
* `quickstart_git_ref` — branch or tag used in every `.../blob/{ref}/...` link in the lab nav pages

When you cut a quickstart release, set `quickstart_git_ref` to that tag (for example `v1.2.0`), then update the **Quickstart Repo** tab URL in `ui-config.yml` and the `page-links` entry in the same `antora.yml` so the `/tree/...` segment matches. Antora does not expand attributes inside those YAML URL strings.

On RHDP, you can override `quickstart_git_ref` via Showroom `user_data` (same key) so a catalog item points at a different tag without forking this repo—see the https://rhpds.github.io/showroom_template_nookbag/modules/user-data.html[user data] flow for your deployment.

== Quick Start

. Clone this repo and preview locally:
+
[source,sh]
----
podman run --rm --name antora -v $PWD:/antora:z -p 8080:8080 -i -t ghcr.io/juliaaano/antora-viewer
----
+
(On non-SELinux systems, omit `:z`.)

NOTE: Mermaid diagrams need `node_modules` from `npm ci`. If diagrams are blank, use `make run` (runs `npm ci` before Podman) or run `npm ci` once in the repo root, then use the command above.

. Edit content in `content/modules/ROOT/pages/`

== Local Development

*Option 1 — single run:*
[source,sh]
----
podman run --rm --name antora -v $PWD:/antora:z -p 8080:8080 -i -t ghcr.io/juliaaano/antora-viewer
----

*Option 2 — podman-compose (live reload):*
[source,sh]
----
podman-compose up
----

Open http://localhost:8080 after the build completes.

*Makefile (local):* `make run` — Podman viewer (runs `npm ci` first so Mermaid resolves); `make check` / `make build` — Antora to `www/`; `make serve` — build then Python `http.server`; `make install` then `make lint` — yamllint + Vale on modules; `make format` — Prettier on playbook YAML. See `make help`. Vale styles come from `vale sync` (`make install` / `make vale-sync`); needs network once.
