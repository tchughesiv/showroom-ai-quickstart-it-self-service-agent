== What You'll Learn

This hands-on lab walks you through the **IT Self-Service Agent** for IT process automation on Red Hat OpenShift. Your **environment is already provisioned**. To open the test email application, stay in Showroom: use the **Test email UI** tab in the right-hand pane (split screen with this lab), or follow the direct link on xref:02-interact.adoc[Interact with the Agent]. You do not need to look up application URLs in RHDP lab output.

By the end of this workshop you will:

* Understand how routing, specialist agents, knowledge bases, and MCP integrations fit together
* Complete the **laptop refresh** flow via the test email UI
* Know where to extend the platform and how evals, tracing, and the blog series support production thinking

*Time to complete:* about 15–25 minutes, depending on how much you read and how long model responses take in the test email UI

== What the IT Self-Service Agent Does

The quickstart implements a **laptop refresh** IT process. Employees can:

* Check their current laptop and eligibility for refresh
* Browse available laptop options by region
* Submit a refresh request that creates a ServiceNow ticket

The platform integrates with knowledge bases, MCP (Model Context Protocol) servers, and external systems like ServiceNow. The same framework can extend to other IT processes: PIA, RFP generation, access requests, and more.

== Key Technologies

* *Llama Stack* — AI inference for Llama models
* *LangGraph* — State machine for agent conversations
* *MCP Servers* — Interface to external systems (e.g. ServiceNow)
* *RAG Knowledge Bases* — Policy and process retrieval
* *OpenTelemetry* — Distributed tracing
* *OpenShift / Kubernetes* — Deployment platform
