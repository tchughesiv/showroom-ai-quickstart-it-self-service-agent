== What You'll Learn

This hands-on lab guides you through deploying and using an AI-driven self-service agent for IT process automation on Red Hat OpenShift.

By the end of this workshop you will:

* Deploy the self-service agent platform to OpenShift using Helm
* Interact with the laptop refresh agent through the CLI
* Run evaluations to validate agent quality
* Use distributed tracing to understand agent workflows

*Time to complete:* 60–180 minutes (depending on deployment mode and optional modules)

== What the Self-Service Agent Does

The quickstart implements a **laptop refresh** IT process. Employees can:

* Check their current laptop and eligibility for refresh
* Browse available laptop options by region
* Submit a refresh request that creates a ServiceNow ticket

The platform integrates with knowledge bases, MCP (Model Context Protocol) servers, and external systems like ServiceNow. The same framework can be extended to other IT processes: PIA, RFP generation, access requests, and more.

== Key Technologies

* *Llama Stack* — AI inference for Llama models
* *LangGraph* — State machine for agent conversations
* *MCP Servers* — Interface to external systems (e.g. ServiceNow)
* *RAG Knowledge Bases* — Policy and process retrieval
* *OpenTelemetry* — Distributed tracing
* *OpenShift / Kubernetes* — Deployment platform

== Where to Start

This lab uses **Demo Mode** deployment (`make install INSTALL_MODE=demo`) — no Knative operators required.

If you have not yet reviewed requirements and setup, start with xref:02-details.adoc[Requirements and Setup]. If your environment is ready, proceed to xref:03-module-01.adoc[Module 1: Deploy to OpenShift].
