== Learning Objectives

By the end of this module you will be able to:

* Open the webmail UI (deployed automatically with Demo Mode)
* Complete the laptop refresh workflow via email
* Test different user scenarios using the user dropdown

== Step 1: Open the Webmail UI

Demo Mode deploys a test email server (Greenmail) with a webmail UI. At the end of Module 1, the deployment displayed a **webmail URL**. Open that URL in your browser.

If you need to find it:

[source,bash]
----
oc get routes -n $NAMESPACE | grep -i email
----

**Expected outcome:** Webmail UI opens with a user dropdown and message composer.

== Step 2: Start the Laptop Refresh Flow

. Select a user from the dropdown (e.g. `alice.johnson@company.com`)
. The subject is pre-filled with "Laptop refresh"
. In the message box, type: `Hi, I'd like to start a laptop refresh request`
. Click "Send"

**Expected outcome:** Agent responds with laptop info, eligibility summary, and asks if you'd like to see available options.

== Step 3: Complete the Workflow

Continue the conversation in the webmail UI:

* **You:** `I'd like to see available laptop options`
* **Expected:** Agent lists four laptop options with specs and pricing for your region (NA, EMEA, APAC, or LATAM)

* **You:** `I'd like option 1, the Apple MacBook Air M2`
* **Expected:** Agent confirms and asks for approval to create a ServiceNow ticket

* **You:** `Yes, please create the ticket`
* **Expected:** Ticket created, ticket number shown (e.g. REQ3143724), confirmation message

== Step 4: Test Different User Scenarios

Select different users from the dropdown to see region and eligibility differences:

* `alice.johnson@company.com` — NA region, eligible for refresh
* `maria.garcia@company.com` — LATAM region, different laptop options
* `john.doe@company.com` — may not be eligible (laptop too new)

**Expected outcome:** Different laptop options by region; different eligibility results by laptop age.

== Resetting Conversation State

To start a fresh conversation with the same user, type `reset` in the message box and send; then send any message (e.g. `hello`) to begin again. This clears conversation history for that user.
