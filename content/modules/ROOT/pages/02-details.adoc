== Prerequisites

=== LLM Access

The quickstart requires access to a Llama 3 70B–capable model. For Demo Mode, use *Model as a Service* via `LLM_URL` and `LLM_API_TOKEN` — point to your existing LLM endpoint (OpenShift AI, cloud API, or similar). This keeps the cluster focused on the agent platform; no on-cluster LLM resources needed.

*If you do not have an endpoint* and must deploy on-cluster: the https://github.com/rh-ai-quickstart/ai-architecture-charts/tree/main/llm-service[LLM Service Helm Chart,window="_blank"] or https://github.com/rh-ai-quickstart/it-self-service-agent/blob/main/docs/alternate-journeys/scout-only.md[Scout-only journey,window="_blank"] add significant hardware on top of the agent requirements below — see those docs for model-specific resources.

=== Hardware Requirements

*Agent platform only* (LLM via `LLM_URL`):

* CPU: 10–12 cores
* Memory: 10 Gi
* Storage: 50 Gi
* *Optional:* GPU for Llama Guard 3 safety shields

=== Software Requirements

* **Helm** — Kubernetes package manager
* **oc CLI** — OpenShift (includes kubectl) or **kubectl** — vanilla Kubernetes
* **git** — Version control
* **make** — Build automation
* *Optional:* **Podman** or Docker — only if building images from source
* *Optional:* **Python 3.12+** and **uv** — only for evaluations or building from source

=== Cluster Requirements

* **Demo Mode** (used by this showroom): OpenShift/Kubernetes, mock eventing, no Knative; adds Greenmail (test email server)
* **Testing Mode:** Same baseline — mock eventing, no Knative; no Greenmail (for CI/dev)
* **Production Mode:** OpenShift 4.17.0+ with:
** OpenShift AI
** Serverless Operator
** Streams for Apache Kafka Operator
** Knative Eventing with Kafka broker

=== Permissions

* Namespace admin in the target OpenShift project
* Ability to pull images from quay.io
* LLM API endpoint with credentials
* *Optional:* Slack, ServiceNow, or email for integrations

== Pre-Workshop Checklist

□ **OpenShift access** — Test `oc login` and `oc whoami`
□ **LLM endpoint** — API URL and token ready
□ **Repository cloned** — `rh-ai-quickstart/it-self-service-agent` (dev branch for now; main when ready)
□ **Tools installed** — Helm, oc, kubectl (Podman only if building images)
□ **Namespace ready** — `make install` creates it; or run `oc new-project my-agent-namespace` first if preferred

== Troubleshooting

=== Demo Mode — Specific

**Problem:** Webmail URL not displayed after `make install`
→ **Solution:** Run `oc get routes -n $NAMESPACE | grep -i email` to find the test-email-server route. Ensure Greenmail pod is Running: `oc get pods -n $NAMESPACE | grep test-email`.

**Problem:** Greenmail or test-email-server pod not ready
→ **Solution:** Check logs: `oc logs -n $NAMESPACE deployment/test-email-server -c greenmail`. Ensure cluster has ~10 Gi allocatable memory. Wait a few minutes — Greenmail can take 2–3 minutes to start.

**Problem:** Agent doesn't respond in webmail
→ **Solution:** Verify LLM connectivity from the cluster: `LLM_URL` and `LLM_API_TOKEN` must be set before `make install`. Check agent-service and request-manager pods: `oc logs -n $NAMESPACE deployment/self-service-agent-service`.

=== Common Issues

**Problem:** `oc command not found`
→ **Solution:** Install the OpenShift CLI from the Red Hat Customer Portal or your cluster's web console.

**Problem:** `LLM connection fails`
→ **Solution:** Verify `LLM_URL` and `LLM_API_TOKEN`. Ensure the endpoint is reachable from the cluster.

**Problem:** `Image pull errors`
→ **Solution:** Confirm quay.io access and image pull secrets. Run `oc get secret` in your namespace.

**Problem:** `Knative/Kafka pods not ready` (Production Mode)
→ **Solution:** See the quickstart’s deployment guide for operator ordering and broker configuration.
